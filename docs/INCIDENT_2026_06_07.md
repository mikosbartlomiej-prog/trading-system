**Status:** Diagnosed + structural fixes shipped (v3.22.0). System remains paper-only, EDGE_GATE disabled, live trading blocked.

## Timeline

- **2026-06-05** 13:45 UTC — Allocator's 8 BUYs ALL rejected by Alpaca. execution.json reason: `"Alpaca rejected order (see stdout)"` — no diagnostic detail. Equity $94,710 (-1.2%).
- **2026-06-05 → 2026-06-07** — LLM Senior PM unavailable 3 consecutive days. Deterministic adapter only. Equity drift to $89,533 → $89,703 (cumulative -4.27% / -$8,744 unrealized).

## Root causes (4 independent)

### 1. Bare "see stdout" rejection reason — *no* structured diagnostic

`shared/allocator.py::_exec_buy` returned `"Alpaca rejected order (see stdout)"` on failure with zero structured fields. Operator cannot diagnose 8 simultaneous failures without scraping runner logs.

**Fix:** `shared/order_rejection_audit.py` + allocator wire-in. Now every rejection carries `rejection_category` (8 enum values), `http_status`, `alpaca_error_code`, `alpaca_message`, `order_notional`, `order_qty`, plus an audit JSONL line `V322_ORDER_REJECTION`.

### 2. No buying-power pre-check before batch execution

8 BUYs of ~$10k each = ~$80k notional vs ~$50k available BP (positions from 2026-06-04 still consuming margin). Allocator submitted all 8 anyway; Alpaca 403'd each.

**Fix:** `shared/allocator_bp_guard.py` — pre-execution batch gate. Walks BUYs in EXIT→REDUCE→BUY priority, drops the tail whose cumulative notional would exceed available BP or the exposure cap. Deferred orders surface in execution.json with `status="deferred_bp"`. Fail-soft when account data unavailable. Cannot raise risk limits (caller-clamp invariant).

### 3. crypto-monitor wiring gap to opportunity ledger

`crypto-monitor/monitor.py` did not import or call `signal_opportunity_ledger.record_opportunity` for ANY decision — accepted, rejected, or observed. BTC RSI 7.6 → strategy noop → no record on disk.

**Fix:** 13 fail-soft `_emit_opportunity()` call sites added across the monitor's decision points. Synthetic test verifies BTC RSI extreme oversold + ETH RSI extreme oversold both write at least one opportunity entry, AND the monitor does not crash if the ledger module is missing, AND alpaca_orders.place_* is never called during signal evaluation.

### 4. LLM unavailable + 14-day override lock = deterministic-only adapter stuck

LLM Senior PM unavailable 3 consecutive days. Existing deterministic adapter logged `"LLM unavailable (skipped) — deterministic adapter only"` and proceeded, but the 14-day LLM override lock (active since 2026-05-30) prevented even legitimate SILENT-strategy adjustments. crypto-momentum was kept enabled but deterministic adapter could not re-evaluate based on RSI 7.6 vs the 62-day silence.

**Fix:** `shared/llm_availability.py` + `runtime_state.json::llm_availability` section. Tracks consecutive failures. After 2 → enqueues `REVIEW_LLM_OUTAGE` (P0). After 3 + SILENT strategy + active override lock → enqueues `REVIEW_SILENT_STRATEGY_LOCK` (P1). **NEVER auto-clears the LLM override lock** — operator-only action.

## Drawdown context — paper unrealized loss

`learning-loop/analyzer.py::compute_equity_gap_alert` already surfaces equity-gap WARN, but the existing daily_drawdown_guard is closed-trade-based. Unrealized -4.27% slipped through.

**Fix:** `shared/drawdown_escalation.py` — advisory layer over unrealized P&L. Maps drawdown to one of NONE / WARN / RESTRICT / EMERGENCY_REVIEW with thresholds at -3% / -5% / -8%. Enqueues operator review actions. Does NOT auto-close anything. RESTRICT level flags `runtime_state.json::drawdown_state.new_entry_restricted=true` for the allocator to read on its next run. The 2026-06-07 incident replays cleanly to ALERT_WARN (-4.266%).
