That is 18 calendar days with zero P&L movement and zero closed trades. I traced this to two independent, confirmed failures — not just "no signals today," but two live bugs:

### 1. `price-monitor/monitor.py` — confirmed dead code path (f-string bug)

`check_long_signal()` (line 239) and `check_short_signal()` (line 313) both contain:

```python
f"...RSI={rsi:.1f if rsi else 'N/A'}..."
```

This is **not valid Python** — the text after `:` in an f-string is parsed as a literal format spec, not a Python expression, so `.1f if rsi else 'N/A'` is passed straight to `format()`. I reproduced it directly:

```
>>> f'RSI={rsi:.1f if rsi else "N/A"}'
ValueError: Invalid format specifier '.1f if rsi else "N/A"' for object of type 'float'
```

Both functions wrap their body in `except Exception as e: print(f"{ticker} ... error: {e}"); return None`, so **every single ticker check raises this ValueError and silently returns `None`, on every run, for both the LONG and SHORT scanners.** This is not a "no signal today" — it is a 100%-of-the-time crash swallowed by a broad except. Net effect: price-monitor has not been able to produce a single LONG or SHORT signal since this code was last touched. (Yesterday's brief commit message already flagged this as "identical to the 2026-05-05 bug" — it is still present in the code today, unfixed.)
